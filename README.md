Generate_Setup_0.1.1.exe  用于生成音乐片段的程序（基于RNN）
Continue_Setup_0.1.1.exe   用于延续音乐片段的程序（基于RNN）

Evaluation_Design_v1.pdf   评估方法总结
对比模型-数据集-标签.pdf    找到的数据集和对比模型

《EMOPIA+》   要用的数据集（包含Q1 Q2 Q3 Q4）四种情绪，高昂（积极+高唤起） 悲痛（消极+高唤起） 轻松（积极+低唤起） 忧郁（消极+低唤起）  

《lstm生成音乐》   包含训练代码以及训练集、生成代码（这里训练只用到了音符，但是没有把音长拍子这些也用于训练，所以训练出来的模型预测的时候音符是等长的，目前先做到这一步吧）

MusicGen.py  目前的音乐生成agent代码

方案一：（首选）
目标（暂定只管音符不管节拍）：

将数据集用音乐工具解析，然后丢进模型训练，训练完，用于生成音乐。

然后agent这边也一样，把解析的数据（只包含音符），放到知识库里面，通过写prompt编排agent工作流，结合rag来生成音乐。

最后，将上述二者生成的音乐，用确定的评估方法进行评估。

方案二：
目标：

不进行模型训练，直接用Generate_Setup_0.1.1.exe 用于生成音乐，Continue_Setup_0.1.1.exe用于延续音乐(但是这个做法可能就导致训练数据不一致，虽然我们不用训练)

然后agent这边也一样，把解析的数据（包含解析后的音符以及拍子等等音乐信息），放到知识库里面，通过写prompt编排agent工作流（eg:分析参考片段 A 和 B。计算它们的平均音程跨度、节奏切分规律以及常用和弦外音。请输出一个简短的‘作曲公式')，结合rag来生成音乐。

最后，将上述二者生成的音乐，用确定的评估方法进行评估。
